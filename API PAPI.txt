Eventos Disponíveis
Lista de todos os eventos que podem ser configurados no webhook:

 Mensagens
Evento	Descrição
messages	Mensagens recebidas (texto, imagem, vídeo, áudio, documento, localização, contato, sticker)
message_status	Status de mensagens enviadas (2=enviado, 3=entregue, 4=lido)
message_reaction	Reações em mensagens (emoji adicionado/removido)
 Grupos
Evento	Descrição
group_update	Atualizações de grupos (nome, foto, descrição alterados)
group_participants	Participantes de grupos (entrada, saída, promoção, rebaixamento)
 Contatos & Presença
Evento	Descrição
presence	Presença de contatos (available, unavailable, composing, recording, paused)
contacts	Atualizações de contatos (nome, foto de perfil)
 Chamadas
Evento	Descrição
call	Chamadas de voz/vídeo (ringing, accepted, rejected, timeout)
 Outros
Evento	Descrição
chats	Atualizações de conversas (arquivado, fixado, silenciado)
labels	Labels/etiquetas (associação e edição)
history_sync	Sincronização de histórico (quando conecta)
all	Receber TODOS os eventos
Headers enviados em cada webhook
 Copiar
Content-Type: application/json
X-Instance-ID: minha-instancia
X-Event-Type: messages | message_status | message_reaction | group_update | ...

 Mídia em Base64 NEW
Quando uma mensagem com mídia (imagem, vídeo, áudio, documento, sticker) é recebida, a API automaticamente faz download e converte para Base64, incluindo no payload do webhook:

 Copiar
// Mensagem com mídia (imagem, vídeo, áudio, documento, sticker)
{
  "type": "message",
  "instanceId": "minha-instancia",
  "data": {
    "key": { "remoteJid": "5511999999999@s.whatsapp.net", "id": "ABC123" },
    "message": { 
      "imageMessage": { 
        "mimetype": "image/jpeg",
        "caption": "Olha essa foto!"
      } 
    },
    "pushName": "João",
    "messageTimestamp": 1702500000,
    // ⬇️ MÍDIA EM BASE64 (automático)
    "media": {
      "base64": "iVBORw0KGgoAAAANSUhEUgAA...",
      "mimetype": "image/jpeg",
      "filename": "file_1702500000.jpeg"
    }
  }
}
Como decodificar Base64
 Copiar
// Node.js - Salvar arquivo
const fs = require('fs');
const { base64, filename } = data.media;
const buffer = Buffer.from(base64, 'base64');
fs.writeFileSync(`./downloads/${filename}`, buffer);

// Python - Salvar arquivo
import base64
file_bytes = base64.b64decode(data['media']['base64'])
with open(f"./downloads/{data['media']['filename']}", 'wb') as f:
    f.write(file_bytes)

// PHP - Salvar arquivo
$fileContent = base64_decode($data['media']['base64']);
file_put_contents("./downloads/{$data['media']['filename']}", $fileContent);
 Nota: Mídias de newsletters/canais não são processadas (não possuem mediaKey para descriptografia).

Storage (PostgreSQL + Redis)
v1.3
A Pastorini API suporta 3 tipos de armazenamento para sessões, configuráveis via variáveis de ambiente:

Tipo	Descrição	Recomendado para
file	Armazena sessões em arquivos no disco	Desenvolvimento, até 50 instâncias
postgres	Armazena sessões no PostgreSQL	Produção, 50-500 instâncias
postgres+redis	PostgreSQL + Redis como cache	Alta escala, 500+ instâncias
 Copiar
# Variáveis de ambiente (.env)

# Tipo de storage: file | postgres | postgres+redis
STORAGE_TYPE=postgres+redis

# PostgreSQL
POSTGRES_HOST=localhost
POSTGRES_PORT=5432
POSTGRES_DB=pastorini_api
POSTGRES_USER=postgres
POSTGRES_PASSWORD=sua_senha

# Redis (apenas para postgres+redis)
REDIS_HOST=localhost
REDIS_PORT=6379
REDIS_PASSWORD=
REDIS_DB=0
Arquitetura com Redis: O Redis atua como cache L1 (in-memory) enquanto o PostgreSQL é o storage persistente. Isso proporciona leituras extremamente rápidas (~1ms) com a segurança de persistência do PostgreSQL.

 Setup Rápido com Docker
Execute um único comando para subir PostgreSQL e Redis:

# Subir PostgreSQL + Redis
npm run setup

# Ou usando docker-compose diretamente
docker-compose up -d

# Iniciar a API
npm run server

# Ou tudo junto (setup + server)
npm run dev

Proxy
Configure proxies HTTP, HTTPS ou SOCKS para suas instâncias. Útil para contornar bloqueios de IP ou usar IPs de diferentes regiões.

POST
/api/instances/:id/proxy
NOVO
Configura um proxy específico para uma instância. Após configurar, reconecte a instância para aplicar.

Parâmetro	Tipo	Descrição
enabled *	boolean	Habilita ou desabilita o proxy para esta instância
url	string	URL do proxy no formato: protocolo://usuario:senha@host:porta
Protocolos Suportados
Protocolo	Exemplo
HTTP	http://user:pass@proxy.example.com:8080
HTTPS	https://user:pass@proxy.example.com:8080
SOCKS4	socks4://proxy.example.com:1080
SOCKS5	socks5://user:pass@proxy.example.com:1080
 Copiar
// Requisição - Habilitar proxy HTTP
POST /api/instances/minha-instancia/proxy
{
  "enabled": true,
  "url": "http://user:pass@proxy.example.com:8080"
}

// Requisição - Habilitar proxy SOCKS5
POST /api/instances/minha-instancia/proxy
{
  "enabled": true,
  "url": "socks5://user:pass@proxy.example.com:1080"
}

// Requisição - Desabilitar proxy
POST /api/instances/minha-instancia/proxy
{
  "enabled": false,
  "url": ""
}

// Resposta
{
  "success": true,
  "proxy": {
    "enabled": true,
    "url": "http://user:pass@proxy.example.com:8080"
  },
  "message": "Proxy configurado. Reconecte a instância para aplicar."
}
GET
/api/instances/:id/proxy
Obtém a configuração de proxy de uma instância e informações sobre o proxy global.

 Copiar
// Requisição
GET /api/instances/minha-instancia/proxy

// Resposta
{
  "proxy": {
    "enabled": true,
    "url": "http://user:pass@proxy.example.com:8080"
  },
  "globalProxy": {
    "enabled": false,
    "url": ""
  }
}
GET
/api/proxy/global
Obtém a configuração do proxy global (definido via variáveis de ambiente).

Variáveis de Ambiente
Variável	Descrição	Exemplo
PROXY_ENABLED	Habilita o proxy global	true ou false
PROXY_URL	URL do proxy global	http://proxy.example.com:8080
 Copiar
// Configuração no .env
PROXY_ENABLED=true
PROXY_URL=http://user:pass@proxy.example.com:8080

// Requisição
GET /api/proxy/global

// Resposta
{
  "globalProxy": {
    "enabled": true,
    "url": "http://user:pass@proxy.example.com:8080"
  }
}
 Prioridade: O proxy configurado na instância tem prioridade sobre o proxy global. Se ambos estiverem habilitados, o da instância será usado.

Webhook
Receba notificações em tempo real sobre mensagens, status de conexão e outros eventos.

POST
/api/instances/:id/webhook
NOVO
Configura a URL do webhook para uma instância.

Parâmetro	Tipo	Descrição
url *obrigatório	string	URL que receberá os eventos (POST)
enabled *obrigatório	boolean	Habilita ou desabilita o webhook
 Copiar
// Requisição
{
  "url": "https://meu-sistema.com/webhook",
  "enabled": true
}
GET
/api/instances/:id/webhook
Retorna a configuração atual do webhook.

Payload dos Eventos
Estrutura do JSON enviado para sua URL.

 Copiar
{
  "event": "messages.upsert",
  "instanceId": "minha-instancia",
  "data": {
    "message": {
      "id": "3EB0...",
      "pushName": "João Silva",
      "from": "5511999999999",          // Identificador principal (Telefone ou LID)
      "phone": "5511999999999",         // Número de telefone limpo (sem sufixos)
      "deviceId": 0,                    // ID do dispositivo (0=Celular, >0=Web/Desktop)
      "to": "5511888888888",
      "isGroup": false,
      "type": "text",
      "body": "Olá, tudo bem?",
      "timestamp": 1706900000
    }
  }
}
 Nota sobre Identificadores:
- phone: Use este campo para salvar o contato. Ele contém apenas o número (DDI+DDD+Num).
- deviceId: Informa qual dispositivo enviou a mensagem (ex: 43 para Web). Útil para logs, mas não deve ser usado para identificar o usuário.

POST
/api/instances/:id/webhook
NOVO
Configura um Webhook Secundário dedicado para IA (opcional). Útil para separar o tráfego de mensagens para bots (n8n, Flowise, Typebot) do sistema principal.

Parâmetro	Tipo	Descrição
aiUrl	string	URL do webhook da IA
aiEnabled	boolean	Habilita o envio para a IA
aiEvents	string[]	Lista de eventos específicos para a IA (ex: ["messages.upsert"]).
 Copiar
// Exemplo: Configurar Webhook Principal + Webhook IA
{
  // Principal (ERP/CRM) - Recebe tudo
  "url": "https://meu-sistema.com/webhook",
  "enabled": true,
  "events": ["all"],

  // IA (n8n/Typebot) - Recebe apenas mensagens
  "aiUrl": "https://n8n.webhook/chat",
  "aiEnabled": true,
  "aiEvents": ["messages.upsert"]
}

Fila de Mensagens
Sistema de fila com Redis para envio de mensagens com rate limiting, retry automático e persistência. Configurável via variáveis de ambiente.

🚀 Como Funciona
A fila processa mensagens automaticamente respeitando o rate limit configurado. Mensagens que falham são reenfileiradas até o máximo de tentativas.

📬
Enfileirar
Mensagem entra na fila
⏱️
Rate Limit
Respeita X msg/min
🔄
Retry
Tenta novamente se falhar
✅
Enviado
Estatísticas atualizadas
GET
/api/queue/status
Verifica se a fila está habilitada e retorna a configuração atual.

 Copiar

  "success": true,

      "messagesPerMinute": 20,
      "maxRetries": 3,
      "retryDelayMs": 5000,
      "processingIntervalMs": 1000
    }
  }
}

⚙️ Variáveis de Ambiente
Configure a fila no arquivo .env:

Variável	Padrão	Descrição
QUEUE_ENABLED	true	Habilita ou desabilita o sistema de fila
QUEUE_RATE_LIMIT	20	Máximo de mensagens por minuto por instância
QUEUE_MAX_RETRIES	3	Máximo de tentativas antes de marcar como falha
QUEUE_RETRY_DELAY_MS	5000	Delay entre tentativas em milissegundos
QUEUE_PROCESSING_MS	1000	Intervalo de processamento da fila em ms
QUEUE_DEBUG	false	Habilita logs detalhados da fila
Exemplo de Configuração
 Copiar
# .env
QUEUE_ENABLED=true
QUEUE_RATE_LIMIT=20
QUEUE_MAX_RETRIES=3
QUEUE_RETRY_DELAY_MS=5000
QUEUE_PROCESSING_MS=1000
QUEUE_DEBUG=false

📢 Canais (Newsletter)
NOVO
Endpoints para gerenciar canais do WhatsApp (Newsletter). Crie, siga e gerencie canais de difusão.

POST
/api/instances/{id}/newsletter/create
Cria um novo canal do WhatsApp.

name	string	obrigatório	Nome do canal
description	string	opcional	Descrição do canal
picture	base64	opcional	Imagem do canal em base64
 Copiar
{
  "name": "Meu Canal de Notícias",
  "description": "Atualizações diárias sobre tecnologia"
}
POST
/api/instances/{id}/newsletter/{jid}/follow
Segue um canal do WhatsApp.

POST
/api/instances/{id}/newsletter/{jid}/unfollow
Deixa de seguir um canal.

GET
/api/instances/{id}/newsletter/{jid}
Obtém informações de um canal.

GET
/api/instances/{id}/newsletter/subscribed
Lista todos os canais que você está seguindo.

GET
/api/instances/{id}/newsletter/{jid}/messages
Obtém mensagens de um canal.

count	int	opcional	Quantidade de mensagens (padrão: 10)
before	string	opcional	ID da mensagem para paginação
POST
/api/instances/{id}/newsletter/{jid}/mute
Silencia ou ativa notificações de um canal.

 Copiar
{ "mute": true }

NOVO
Endpoints para gerenciar configurações de privacidade.

/api/instances/{id}/privacy-settings

Resposta:
 Copiar
{
  "lastSeen": "contacts",
  "profilePhoto": "all",

  "readReceipts": "all",
  "groupAdd": "contacts",
  "online": "all",
  "callAdd": "all"
}
POST
/api/instances/{id}/privacy-settings
Atualiza uma configuração de privacidade.

setting	string	obrigatório	lastSeen, profilePhoto, status, readReceipts, groupAdd, online, callAdd
value	string	obrigatório	all, contacts, contact_blacklist, match_last_seen, none

{

  "value": "contacts"
}
